### 第二部分：演示场景1 - 完整信息输入（3分钟）

**话术：**
> "首先演示最标准的场景，用户一次性给出完整信息。"

**输入示例1：**
```
请帮我开票：给腾讯开软件服务，金额50000元，数量5个，单价10000元/个
```

**预期效果：**
1. 系统显示"收到，正在提取开票要素..."（2-4秒）
2. 成功解析并显示绿色成功气泡
3. 自动补全腾讯的完整企业信息：
   - 全称：深圳市腾讯计算机系统有限公司
   - 税号：91440300708461136T
   - 地址、电话、开户行等
4. 自动匹配软件服务的6%税率
5. 计算税额：50000 × 6% = 3000元
6. 显示价税合计：53000元
7. 右侧实时显示完整发票

**讲解重点：**
- ✅ 预置引导：输入框有明确提示
- ✅ 自然语言解析：从一句话中提取5大字段
- ✅ 智能补全：自动识别"腾讯"并补全全部企业信息
- ✅ 税率匹配：根据"软件服务"自动匹配6%
- ✅ 自动计算：税额和价税合计

---

### 第三部分：演示场景2 - 逻辑校验（2分钟）

**话术：**
> "接下来演示系统的智能校验功能。如果用户输入的数据逻辑不对，系统会立即提示。"

**输入示例2（故意错误）：**
```
给阿里开云服务，单价100元，数量5个，总共1000元
```

**预期效果：**
1. 系统显示红色错误气泡
2. 提示：**"逻辑校验不通过：5 × 100 = 500 ≠ 1000，请修正"**
3. 右侧发票预览不更新

**讲解重点：**
- ✅ 实时校验：数量 × 单价 必须等于金额
- ✅ 错误提示：清晰告知用户哪里有问题
- ✅ 防止错误：不会生成逻辑错误的发票

**修正输入：**
```
给阿里开云服务，单价100元，数量5个，总共500元
```

**预期效果：**
- 成功解析并生成发票
- 自动补全阿里巴巴企业信息
- 税率6%，税额30元，价税合计530元

### 第四部分：演示场景3 - 自动计算（2分钟）

**话术：**
> "系统还支持部分信息输入，自动计算缺失的数据。"

**输入示例3：**
```
给字节开技术服务，数量10个，单价15000元
```

**预期效果：**
1. 系统自动计算金额：10 × 15000 = 150000元
2. 补全字节跳动企业信息
3. 匹配技术服务6%税率
4. 计算税额：9000元
5. 价税合计：159000元

**讲解重点：**
- ✅ 智能计算：只需提供两个值，自动算第三个
- ✅ 减少输入：提升用户体验
- ✅ 历史模板：字节跳动的技术服务常用单价15000元（已预置）

### 第五部分：演示场景4 - 货物销售（2分钟）

**话术：**
> "系统不仅支持服务类发票，也支持货物销售，税率自动切换。"

**输入示例4：**
```
给华为开硬件设备，金额26000元，数量2台，单价13000元
```

**预期效果：**
1. 成功解析并补全华为企业信息
2. 识别"硬件设备"属于货物销售

5. 价税合计：29380元

**讲解重点：**
- ✅ 税率智能匹配：
  - 现代服务类（软件、技术、咨询、云服务）→ 6%
  - 货物销售类（电子产品、硬件、办公用品）→ 13%
- ✅ 自动分类：发票上显示"货物销售"类别

## 🎨 演示亮点总结

### 1. 预置引导
- 输入框Ghost Text清晰提示需要哪些信息
- 降低用户学习成本

### 2. 自然语言解析（NLU）
- 支持多种表达方式
- 自动提取5大关键字段：
  - ✓ 收票方（客户名称）
  - ✓ 金额
  - ✓ 数量
  - ✓ 单价
  - ✓ 商品类型

### 3. 逻辑校验
- 实时验证数学关系
- 红色气泡明确提示错误
- 防止生成错误发票

### 4. 智能补全
- **客户信息**：识别腾讯/阿里/字节/华为等，自动补全税号、地址、电话、开户行
- **税率匹配**：根据商品类型自动选择6%或13%
- **自动计算**：只需2个参数即可算出第3个
- **历史模板**：基于历史开票习惯推荐单价

### 5. 实时预览
- 右侧即时显示发票效果
- 符合增值税电子普通发票格式
- 专业的视觉呈现

## 💡 应对可能的问题

### Q1: 如果用户输入的客户不在数据库怎么办？
**A:** 当前演示版会保留用户输入的客户名称，但不会自动补全详细信息。后续版本会：
- 提示用户"该客户为新客户，请补充企业信息"
- 支持多轮对话逐步收集信息
- 自动保存到客户库供下次使用

### Q2: 如果解析不准确怎么办？
**A:** 
- 当前是基于规则的解析，后续会接入大语言模型（如GPT-4、文心一言等）
- 支持用户手动修正，系统学习优化
- 可以多轮对话澄清信息

### Q3: 如何保证数据安全和隐私？
**A:**
- 所有数据传输采用HTTPS加密
- 敏感信息（税号、银行账号）加密存储
- 符合《数据安全法》和《个人信息保护法》要求
- 支持企业私有化部署

### Q4: 能支持专票吗？
**A:** 
- 当前演示版展示的是普票
- 专票需要额外的资质审核和更严格的字段校验
- 技术上完全可以支持，只需调整字段和格式

### Q5: 性能如何？能支持多少并发？
**A:**
- 当前演示版单机可支持100+并发
- 解析速度：< 1秒
- 基于Next.js SSR，可轻松扩展到云服务（Vercel、阿里云等）
- 预计生产环境可支持1000+并发
